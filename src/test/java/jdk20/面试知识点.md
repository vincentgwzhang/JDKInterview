## 常见考题

**1. Scoped Value 和 ThreadLocal 区别？

**答：**
| | ThreadLocal | Scoped Value |
|---|-------------|--------------|
| 绑定 | 线程 | **动态作用域**（调用链） |
| 可变性 | 可变 | **不可变**（绑定后不可改） |
| 继承 | `InheritableThreadLocal` 复杂 | 结构化传播，虚拟线程友好 |
| 清理 | 需手动 `remove()`，易泄漏 | 作用域结束自动失效 |
| 场景 | 线程内可变上下文 | 请求 ID、追踪、权限快照 |

**2. 为什么 ThreadLocal 在虚拟线程场景可能有问题？

**答：**
- 虚拟线程数量可达**百万级**，每个都带 ThreadLocal Map 会**放大内存**（value 生命周期跟线程绑定）。
- 线程池复用平台线程时 ThreadLocal 易**泄漏**；虚拟线程量大时问题更明显。
- **Scoped Value** 按作用域绑定、不可变，更适合「请求级上下文」在大量虚拟线程间传递（JDK25 转正）。
