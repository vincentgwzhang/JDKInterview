# JDK 14 面试要点（2020）

## 1. Records（预览，JDK16 转正）

- `record Point(int x, int y) {}` 自动生成构造器、访问器、`equals/hashCode/toString`
- **不可变**数据载体；可自定义紧凑构造器做校验
- 不能继承类（隐式 final），可实现接口

## 2. instanceof 模式匹配（预览，JDK16 转正）

- `if (obj instanceof String s)` 直接使用 `s`

## 3. Switch 表达式（标准特性）

- 正式成为标准，不再 preview

## 4. 其他

- **Helpful NullPointerExceptions**：指出哪个变量为 null
- **jpackage**（了解）：打包原生安装包

## 常见考题

**1. Record 和 Lombok `@Data` 区别？能加字段吗？

**答：**
| | Record | Lombok `@Data` |
|---|--------|----------------|
| 语义 | 不可变数据载体 | 可变 POJO |
| 生成 | 语言级：构造器、访问器、equals/hashCode/toString | 编译期注解生成 |
| 字段 | **不能**随意加实例字段（仅构造器参数对应组件） | 任意加字段 |
| 继承 | 隐式 `final`，不能继承类 | 可继承 |
| 适用 | DTO、值对象、领域事件 | 实体、需要 setter 的模型 |

**2. Record 如何实现接口、定制 `equals`？

**答：**
- **可实现接口**，可定义 `default` 方法逻辑。
- **不能覆盖** `equals`/`hashCode`/`toString`（由组件自动生成）。
- 可写**紧凑构造器**做校验：`record User(String name) { User { if (name==null) throw ...; } }`
- 可添加**静态方法、静态字段**；实例字段仅来自 record 组件。

**3. 模式匹配 instanceof 如何减少强转样板代码？

**答：**
```java
// 旧
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}
// 新
if (obj instanceof String s) {
    System.out.println(s.length());
}
```
- 编译器保证 `s` 仅在类型匹配的分支内有效，避免重复强转与 `ClassCastException` 风险。
