# JDK 21 面试要点（2023，LTS）⭐

## 1. 虚拟线程（正式）

- `Executors.newVirtualThreadPerTaskExecutor()`
- **Pinning**：synchronized/native 可能固定到载体线程；面试可提 `jvm`-tool 与替代 `ReentrantLock`
- **不要**池化虚拟线程（每个任务一个即可）

## 2. Sequenced Collections

- `SequencedCollection`、`SequencedSet`、`SequencedMap`
- 统一 API：`getFirst`/`getLast`、`addFirst`/`addLast`、`reversed()`

## 3. Pattern Matching for switch（正式）

- 类型模式、`when` 守卫、与 null 处理
- 配合 sealed 穷尽检查

## 4. Record Patterns（正式）

- `case Point(int x, int y)` 解构

## 5. 其他高频

- **String Templates**（预览，JDK23+ 演进，面试了解语法方向）
- **Generational ZGC**
- **Key Encapsulation Mechanism API**（了解）
- **结构化并发**仍预览

## 常见考题

**1. 虚拟线程原理与线程池选型？

**答：**
- **原理**：JVM 调度轻量线程；阻塞时 unmount，carrier 线程可执行其他虚拟线程。
- **选型**：I/O 密集（Web、DB、RPC）→ `Executors.newVirtualThreadPerTaskExecutor()`，**每任务一个虚拟线程，不要池化**。
- CPU 密集 → 平台线程池，并行度 ≈ CPU 核数。
- 避免在热路径长时间 `synchronized`（考虑 `ReentrantLock`）以减少 pin。

**2. `getFirst()` 空集合行为？

**答：** 对**空** `SequencedCollection` 调用 `getFirst()` / `getLast()` 抛 **`NoSuchElementException`**（与 `peekFirst` 等可能返回 null 的 API 区分，以具体方法文档为准）。应先 `isEmpty()` 判断或用 `Optional` 包装逻辑。

**3. Switch 模式匹配中 `when` 子句执行顺序？

**答：**
- 按 **`case` 书写顺序** 匹配；先匹配类型/模式，再评估 **`when` 守卫**。
- 例：`case String s when s.length() > 3` 需在更宽模式之前，否则宽模式先命中。
- 与 `if-else` 链类似，**更具体的分支应靠前**；配合 sealed 类型做穷尽检查。

**4. 为什么 21 是 Java 8 之后最重要的 LTS？

**答：**
- **虚拟线程**正式落地，高并发 I/O 编程模型变革。
- **模式匹配**（switch、record patterns）定稿，与 Records、Sealed 形成现代 Java 风格。
- **Sequenced Collections**、Generational ZGC 等实用 API/JVM 改进。
- Spring Boot 3.2+、主流框架以 21 为新基线；8→21 是下一轮企业升级主路径。
