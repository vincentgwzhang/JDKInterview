# JDK 9 面试要点（2017）

## 1. 模块系统 JPMS（Java Platform Module System）

- **`module-info.java`**：`requires`、`exports`、`opens`、`provides` / `uses`
- **面试角度**：强封装、启动层可维护性；**不是**微服务模块化
- **classpath vs modulepath**：未模块化代码仍在 unnamed module
- 常问：**JDK 9 之后还能反射访问 JDK 内部 API 吗？** 默认不能，需 `--add-opens`

## 2. 集合工厂方法

- `List.of()`、`Set.of()`、`Map.of()` / `Map.ofEntries()`
- **不可变**、拒绝 null、重复 key 抛异常
- 与 `Collections.unmodifiableList` 区别：工厂方法返回真正不可变实现

## 3. 接口私有方法

- 在 `default` 方法间复用逻辑，不暴露给实现类

## 4. try-with-resources 增强

- 可在 try 外声明 effectively final 的资源变量

## 5. Stream 增强

- `takeWhile`、`dropWhile`（有序流）、`ofNullable`

## 6. 其他

- **JShell**：REPL，了解即可
- **Process API** 改进：面试较少
- **G1** 成为默认 GC（了解）

## 常见考题

**1. 模块解决了什么问题？和 Maven 模块一样吗？

**答：**
- **JPMS 解决**：强封装（未 export 的包默认不可访问）、明确依赖、可构造更小运行时镜像（`jlink`）、启动期可靠性。
- **与 Maven 区别**：Maven 是**构建/依赖管理**；JPMS 是 **JVM 运行时** 的模块边界。可配合使用，概念不同。
- **未模块化代码**：仍在 classpath 的 **unnamed module**，可读导出模块。

**2. `List.of(1,2)` 能 `add` 吗？能改元素吗？

**答：**
- **不能** `add`/`remove`，会抛 `UnsupportedOperationException`。
- **不能** `set` 修改元素（`List.of` 返回真正不可变列表）。
- 与 `Collections.unmodifiableList` 不同：后者是**视图**，原列表改则视图也变；`List.of` 是独立不可变副本。
- 不允许 `null` 元素。

**3. `takeWhile` 与 `filter` 在无序流上的区别？

**答：**
- **`takeWhile`**：遇到第一个不满足条件的元素后**停止**处理后续元素（短路）；在**有序流**上像「前缀截取」。
- **`filter`**：会**扫描整个流**，保留所有满足条件的元素。
- **无序流**：`takeWhile` 结果不确定（规范不保证顺序语义），生产环境慎用于无序并行流。
- 例：有序 `[2,4,6,7,8]`，`takeWhile(n%2==0)` → `[2,4,6]`；`filter` 会得到所有偶数。
