# JDK 11 面试要点（2018，LTS）

## 1. 字符串新方法

- `isBlank()`：空或仅空白
- `lines()`：按行 Stream
- `strip()` / `stripLeading()` / `stripTrailing()`：Unicode 空白
- `repeat(int)`

## 2. 标准 HTTP Client（`java.net.http`）

- 替代 HttpURLConnection；支持 HTTP/2、异步
- `HttpClient`、`HttpRequest`、`HttpResponse`

## 3. 文件读写便捷 API

- `Files.readString` / `writeString`（需 `Charset`）

## 4. Optional

- `isEmpty()` 等价于 `!isPresent()`

## 5. 移除与迁移（概念题）

- **Java EE / CORBA 模块移除**（javax.* 需单独依赖）
- **Nest-Based Access Control**：反射可见嵌套类关系
- **ZGC**（实验）、**Epsilon**（实验）GC

## 6. 运行 `java` 单文件源码

- `java Hello.java` 直接运行（无 javac 步骤）

## 常见考题

**1. `strip()` 与 `trim()` 区别？

**答：**
- **`trim()`**：按 **≤ U+0020（ASCII 空格）** 删除首尾字符，不处理全角空格等 Unicode 空白。
- **`strip()`**：按 **Unicode 空白类别** 删除（含全角空格、不间断空格等），更适合国际化文本。
- 另有 `stripLeading()` / `stripTrailing()` 只处理一侧。

**2. HTTP Client 同步与异步调用方式？

**答：**
```java
HttpClient client = HttpClient.newHttpClient();
HttpRequest req = HttpRequest.newBuilder().uri(URI.create(url)).GET().build();
// 同步
HttpResponse<String> res = client.send(req, HttpResponse.BodyHandlers.ofString());
// 异步
CompletableFuture<HttpResponse<String>> fut =
    client.sendAsync(req, HttpResponse.BodyHandlers.ofString());
```
- 支持 HTTP/2、超时、重定向、BodyPublisher/BodyHandler；替代老 `HttpURLConnection`。

**3. JDK 11 为什么很多企业从 8 升级到 11？

**答：**
- **LTS** 长期支持，安全补丁周期明确。
- **移除 Java EE 模块**（Tomcat/Spring 等自带依赖更清晰）。
- **GC 改进**（G1 默认、ZGC 实验）、性能与容器友好性提升。
- **语言/API**：`var`（10）、新 HTTP Client、字符串 API、`Optional.isEmpty` 等。
- 8→11 是业界验证最广的跳跃路径之一，生态（Spring Boot 2.7+ 等）成熟。
