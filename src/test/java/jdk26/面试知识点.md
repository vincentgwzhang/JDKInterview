# JDK 26 面试要点（2026，非 LTS）

> JDK 26 为非 LTS，已在 2026-03-17 GA。面试通常问「方向」和少数已交付 JEP，而不是把它当企业主力版本。

## 1. 语言与 API 持续演进

- **Primitive Types in Patterns, instanceof, and switch**：第四次预览
- **Structured Concurrency**：第六次预览
- **HTTP/3 for the HTTP Client API**：`java.net.http` 继续演进

## 2. 性能与云原生

- Ahead-of-Time Object Caching、G1 吞吐优化等
- **虚拟线程**生态成熟：框架（Spring Boot 3.2+）默认支持

## 3. 兼容性与移除

- **Remove the Applet API**
- **Prepare to Make Final Mean Final**：反射/Unsafe 修改 final 字段会逐步收紧

## 4. 安全

- PEM Encodings of Cryptographic Objects 第二次预览
- 关注 TLS、证书、密码学 API 的发行说明

## 5. 面试回答模板

- 「26 非 LTS，生产仍以 **21/25 LTS** 为主；26 用于验证下一批 JEP」
- 关注 [OpenJDK JDK 26](https://openjdk.org/projects/jdk/26/) 的 Target 26 条目

## 常见考题

**1. 非 LTS 版本是否适合生产？

**答：**
- **一般不建议**核心业务直接上非 LTS（如 26）：支持周期短（约 6 个月），补丁窗口紧。
- **适合**：尝鲜新 JEP、CI 兼容性测试、类库作者验证 forward compatibility。
- **生产推荐**：21 或 **25 LTS**；26 验证通过的功能会进入后续 LTS。

**2. 如何跟踪 Java 新特性？

**答：**
- **[OpenJDK JEP 列表](https://openjdk.org/jeps/0)**：特性状态（Draft / Preview / Target xx）。
- **JDK Release Notes**：每个版本行为变更、移除项。
- **框架发布说明**：Spring Boot、Micronaut 的最低 JDK 要求。
- **Preview 注意**：预览 API 可能跨版本改名或删除，生产需 `--enable-preview` 且承担迁移成本。

**3. 从 21 升级到 25/26 要评估什么？

**答：**
- **依赖字节码版本**：第三方 jar 是否支持 25。
- **预览 API 移除/变更**：是否误用 preview 代码。
- **虚拟线程**：框架（Spring 3.2+）、JDBC 驱动、线程池配置是否 VT-aware。
- **废弃/移除**：Security Manager、部分 GC 参数等。
- **测试**：全量回归 + 压测；灰度发布。
- **结构化并发 / Scoped Value**：若已采用，核对 API 从 preview 到正式的签名变化（尤其 JDK25/26 `StructuredTaskScope`）。
