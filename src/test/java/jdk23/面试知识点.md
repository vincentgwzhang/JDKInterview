## 常见考题

**1. 为什么要在调用 `super()` 前允许构造器逻辑？

**答：**
- 旧规则：构造器第一句必须是 `super()`/`this()`，无法在调用父类构造前做校验或计算参数。
- **灵活构造器体**：允许在 `super(...)` **之前**执行语句（如校验、计算传给父类的参数），例如：
```java
Derived(int x) {
    if (x < 0) throw new IllegalArgumentException();
    super(x * 2);
}
```
- 减少静态工厂方法样板，保证对象创建前不变式已检查。

**2. 原始类型模式匹配与经典 `case 1:` 有何不同？

**答：**
- 经典 `case 1:`：与 **int 值** 比较，无解构。
- **原始类型模式**：在模式匹配 switch 中对 `int`/`long` 等做**类型与范围模式**、与 `when` 守卫组合，与引用类型模式统一语法。
- 编译器可做**穷尽性分析**（配合 sealed/enum）；减少强制转换和重复 `if-else`。
