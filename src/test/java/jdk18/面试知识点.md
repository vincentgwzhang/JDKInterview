# JDK 18 面试要点（2022）

## 1. UTF-8 成为默认 Charset

- `Charset.defaultCharset()` 在 macOS/Windows 等也为 UTF-8（减少平台差异 bug）

## 2. 简易 Web Server（`com.sun.net.httpserver` 工具）

- `jwebserver` 命令或 API 快速静态文件服务（开发/演示用）

## 3. 代码片段 @snippet（API 文档，了解）

- JavaDoc 嵌入可编译示例

## 4. 其他

- **`java.io.File` 反序列化过滤**（安全）
- **Vector API** 第三次孵化
- **Deprecate Finalization**（了解）

## 常见考题

**1. 为什么默认 UTF-8 对跨平台应用重要？

**答：**
- JDK18 前 Windows/macOS 上默认 Charset 常为 **GBK/平台编码**，Linux 常为 UTF-8，读写文件/网络时易出现**乱码**。
- 统一默认 **UTF-8** 后，`FileReader`、`Files.readString` 无参重载等行为跨平台一致，减少「本地能跑、上线乱码」。
- 仍建议 API 显式指定 `StandardCharsets.UTF_8`，不依赖默认值。

**2. 简易 Web Server 适用场景？和生产容器区别？

**答：**
- **适用**：本地静态资源预览、`jwebserver` 快速演示、教学、集成测试 mock 静态站。
- **不适用**：生产 HTTP 服务——无完整安全、集群、监控、反向代理、Servlet 容器生态。
- **与 Tomcat/Jetty 区别**：极简、单线程或简单模型；生产用专业容器 + Spring Boot 等。
