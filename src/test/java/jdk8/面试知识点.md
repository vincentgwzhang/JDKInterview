# JDK 8 面试要点（2014，LTS）

## 1. Lambda 与函数式接口

- **语法**：`(参数) -> 表达式` 或 `-> { 语句块 }`
- **函数式接口**：仅一个抽象方法（`@FunctionalInterface`），如 `Predicate`、`Function`、`Consumer`、`Supplier`
- **方法引用**：`类::静态方法`、`对象::实例方法`、`类::实例方法`、`类::new`
- **面试常问**：Lambda 本质？答：invokedynamic + 生成实现函数式接口的类；捕获的局部变量必须 **effectively final**

## 2. Stream API

- **惰性**：中间操作（`filter`、`map`）不执行，终端操作（`collect`、`forEach`）才触发
- **并行流**：`parallelStream()` 适合 CPU 密集、无共享可变状态；错误共享状态会导致 bug
- **常用**：`map` / `flatMap`、`reduce`、`collect(Collectors.toList())`
- **陷阱**：`forEach` 修改外部变量、在并行流里用 `ArrayList` 非线程安全

## 3. Optional

- 避免 NPE 的容器，**不要**当通用字段类型滥用
- `of`（非 null）、`ofNullable`、`orElse` vs `orElseGet`（后者懒加载）

## 4. 接口默认方法与静态方法

- `default` 解决接口演进；多继承冲突：子类优先，否则显式指定 `Interface.super.method()`
- 静态方法属于接口本身

## 5. java.time（JSR-310）

- `LocalDate` / `LocalDateTime` / `ZonedDateTime` / `Instant` / `Duration` / `Period`
- 不可变、线程安全；替代 `Date` / `Calendar`

## 6. 其他高频

- **CompletableFuture**：异步组合 `thenApply`、`thenCompose`、`allOf`
- **ConcurrentHashMap**：`computeIfAbsent` 等（面试常结合缓存）
- **HashMap**：JDK8 链表转红黑树（阈值 8），与 hash 冲突、resize 相关题

## 常见考题

> 每题下方为 **参考答案**（请向下滚动查看）。

**1. Lambda 和匿名内部类的区别？**

**答：**
- **本质**：匿名类编译为独立 `.class`；Lambda 用 `invokedynamic` + `LambdaMetafactory` 生成函数式接口实现，更轻量。
- **this**：匿名类中的 `this` 指向匿名类自身；Lambda 的 `this` 指向外围类。
- **限制**：Lambda 只能用于函数式接口；捕获的局部变量必须 **effectively final**。
- **字节码**：Lambda 首次执行可能懒生成实现类，后续可缓存。

**2. Stream 中间操作与终端操作？`reduce` 和 `collect` 区别？**

**答：**
- **中间操作**（`filter`、`map`）：返回新 Stream，**惰性**，不触发计算。
- **终端操作**（`collect`、`forEach`、`reduce`）：触发整条流水线执行。
- **`reduce`**：把流聚合成**一个值**（如 sum、max），强调归约。
- **`collect`**：用 `Collector` 聚合成**容器或复杂结构**（List、Map、分组统计），更灵活。

**3. `orElse` 和 `orElseGet` 性能差异？**

**答：**
- `orElse(T other)`：**无论 Optional 是否有值**，`other` 表达式都会先求值再传入。
- `orElseGet(Supplier)`：仅当 **empty** 时才调用 Supplier 生成默认值（懒加载）。
- 若默认值计算昂贵（如查库），应用 `orElseGet`；常量默认值两者皆可。

**4. 接口为什么能有方法体？Java 8 之前如何实现类似扩展？**

**答：**
- **default 方法**：解决接口新增方法导致所有实现类必须改代码的问题；子类继承默认实现，可覆盖。
- **static 方法**：接口上的工具方法，不属于实现类实例。
- **Java 8 之前**：抽象类提供空实现、适配器模式，或第三方（如 Guava）扩展；无法在不破坏兼容的前提下给接口加实例方法。

**5. `HashMap` JDK7 与 JDK8 结构差异？**

**答：**
| | JDK7 | JDK8+ |
|---|------|-------|
| 桶结构 | 数组 + 链表 | 数组 + 链表 / **红黑树** |
| 树化条件 | 无 | 链表长度 ≥8 且数组长度 ≥64 |
| 插入 | 头插（并发扩容曾死循环，已修） | 尾插 |
| hash | 多次扰动 | 高 16 位异或低 16 位 |
| 扩容 | 可能 rehash 死循环（多线程） | 优化，仍非线程安全 |
