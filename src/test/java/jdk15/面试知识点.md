## 常见考题

**1. Sealed 和 final 区别？为什么需要 permits？

**答：**
- **`final`**：禁止任何类继承。
- **`sealed`**：只允许 **`permits` 列出的子类** 继承，介于「完全开放」与「完全封闭」之间。
- **`permits` 作用**：显式声明合法子类型集合，编译器、switch 模式匹配可据此做**穷尽性检查**。
- 子类必须是 `final`、`sealed` 或 `non-sealed`（允许再被任意继承）之一。

**2. 密封类对领域建模、switch 穷尽性有什么帮助？

**答：**
- **领域建模**：表达「结果只有这几种」——如 `sealed interface Result permits Success, Failure`。
- **switch 穷尽**：对 sealed 类型做 `switch` 时，若覆盖所有 permitted 子类型，编译器可验证**无需 default**，新增子类会触发编译提醒补全分支。
- 与 **Record** 组合：代数数据类型风格，减少非法状态和 `instanceof` 链。
