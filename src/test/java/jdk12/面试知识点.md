## 常见考题

**1. Switch 表达式里还能省略 `break` 吗？箭头形式呢？

**答：**
- **Switch 表达式**（有返回值）用 **`->`**：不需要 `break`，右侧表达式即该分支值；多语句用块 `{ yield value; }`（JDK13+ 可用 `yield`）。
- **传统 switch 语句**仍可用 `case: ... break;`。
- **多标签**：`case 1, 2, 3 -> "x";` 合并分支。
- **穷尽性**：表达式作为赋值时，编译器检查是否覆盖所有可能（配合 enum/sealed 更严）。

**2. `teeing` 使用场景？

**答：**
- 对**同一 Stream** 同时做两种收集，再合并结果，**只遍历一次**。
- 典型：同时求 `min` 和 `max`、同时求 `sum` 和 `count`。
```java
stream.collect(Collectors.teeing(
    Collectors.minBy(Comparator.naturalOrder()),
    Collectors.maxBy(Comparator.naturalOrder()),
    (min, max) -> new MinMax(min.orElseThrow(), max.orElseThrow())));
```
- 避免 `collect` 两次开流的成本与一致性风险。
