# JDK 10 面试要点（2018）

## 局部变量类型推断 `var`

- **仅用于局部变量**，不能用于字段、方法参数、返回类型（JDK10 范围）
- 必须有初始化表达式；不能 `var x = null`
- 保留类型名，不是新关键字（`var` 仍可作标识符在旧代码里，但不建议）
- 降低样板代码，**不降低类型安全**（仍是静态类型）

## 常见考题

**1. `var` 是动态类型吗？

**答：** **否**。`var` 是**编译期**根据初始化表达式推断具体类型，字节码里仍是明确类型（如 `ArrayList<String>`），与 JavaScript 等动态类型无关，类型安全不变。

**2. 哪些场景不能用 `var`？

**答：**
- 成员变量、方法参数、方法返回值（JDK10 仅局部变量；后续版本对参数等有扩展，以当前 JDK 为准）。
- 没有初始化表达式：`var x;` 非法。
- 初始化为 `null`：`var x = null` 无法推断类型。
- Lambda 形参需显式类型时（JDK10 的 lambda 参数不能用 var，后续版本已放宽）。
- 会降低可读性的场景（如 `var x = foo()` 看不出类型时慎用）。

**3. `var list = List.of(1); list = List.of("a");` 可以吗？

**答：** **不行**。`var` 推断为 `List<Integer>`，第二次赋值 `List<String>` 类型不兼容，编译错误。`var` 变量类型在声明时即固定。
