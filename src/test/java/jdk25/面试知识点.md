# JDK 25 面试要点（2025，LTS）⭐

## 1. LTS 定位

- 将 JDK 21 以来的预览特性继续收敛：**Scoped Values 转正**，结构化并发进入第五次预览并调整 API
- 企业选型：**21 → 25** 升级路径是面试热点

## 2. 结构化并发（Structured Concurrency，第五次预览）

- `StructuredTaskScope.open(Joiner...)`：JDK 25 起 API 以 **Joiner** 表达「全部成功 / 任一成功 / 等待全部」等策略
- 子任务 `fork`，`join()` 后通过 `Subtask.get()` 取结果；失败时由 Joiner 统一处理
- 对比 `ExecutorService` + 手动 `Future.cancel`

## 3. Scoped Values（正式）

- 请求级上下文、虚拟线程场景替代 ThreadLocal

## 4. 语言特性稳定

- 未命名变量、灵活构造函数体、模块导入等按 JEP 状态转正或继续迭代

## 5. JVM

- 启动与内存占用优化、GC 停顿（概念）
- **Compact Object Headers** 等（了解：减少堆开销）

## 常见考题

**1. 25 与 21 比，面试应强调哪些变化？

**答：**
- **结构化并发**：仍是预览 API，但 `StructuredTaskScope.open(...)` + `Joiner` 让策略表达更清晰。
- **Scoped Values**：替代 ThreadLocal 的请求上下文方案转正。
- **灵活构造器体**、**未命名变量 `_`** 等语言特性成熟。
- **JVM**：Compact Object Headers、启动/内存优化；GC 持续改进。
- 面试话术：21 引入虚拟线程与模式匹配；25 把 Scoped Values 推入标准，并继续打磨结构化并发。

**2. 结构化并发如何保证「要么全成功要么全取消」？

**答：**
- 在 `StructuredTaskScope` 内 `fork` 的子任务绑定到**同一作用域**。
- 使用合适的 `Joiner` 策略：任一子任务失败 → scope 关闭 → **其余子任务被取消**。
- `join()` 后由 Joiner 统一返回结果或传播失败，避免孤儿线程与吞异常。
- 比手动 `Future.get()` + `cancel(true)` 更结构化、可维护。

**3. 为什么很多公司 LTS 策略是 8 → 11 → 17 → 21 → 25？

**答：**
- **风险可控**：每次跳一个 LTS，给依赖库、框架、测试窗口适配时间。
- **支持周期**：Oracle/各发行版对 LTS 提供长期安全补丁；非 LTS 仅短期。
- **生态节奏**：Spring、Kafka 客户端等以 LTS 为基线发布；跟主流减少踩坑。
- **8→11**：去 EE 模块、HTTP Client；**11→17**：Records/Sealed；**17→21**：虚拟线程；**21→25**：Scoped Values 转正、结构化并发 API 继续收敛。
